Key Concepts

These concepts help explain how HTTP works behind the scenes.

1. Client and Server

The client is usually a browser. The server stores and sends the website files.

2. Request

A request is the message sent by the browser asking for a resource such as an HTML page, image, or CSS file.

3. Response

A response is the message sent back by the server. It usually includes the requested content and a status code.

4. Status Codes

Status codes tell the browser what happened. For example, 200 means success and 404 means the page was not found.

5. Headers

Headers provide extra information about requests and responses, such as content type, cache settings, or authentication details.

6. Caching

Caching saves copies of files so pages can load faster instead of downloading everything again.

7. Encryption

Encryption protects data from being read by others while it travels between the browser and server.
